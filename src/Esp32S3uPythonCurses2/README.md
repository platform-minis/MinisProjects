## Glossary

| Term | Description |
| ---- | ----------- |
| **GPIO** | General Purpose Input/Output — a microcontroller pin that can act as a digital input or output |
| **Pin.OUT** | Output mode — the microcontroller drives the voltage (0 V or 3.3 V) |
| **Pin.IN** | Input mode — the microcontroller reads the state of an external signal |
| **SPI** | Serial Peripheral Interface — a fast, full-duplex serial bus using 4 wires (SCK, MOSI, MISO, CS); one master drives multiple slaves, each with its own CS line |
| **SCK / CLK** | SPI clock line — generated by the master (ESP32-S3) to synchronise data transfer |
| **MOSI / DIN** | Master Out Slave In — data sent from the master to the slave device |
| **MISO** | Master In Slave Out — data sent from the slave back to the master; MAX7219 does not use this line |
| **CS / CE / LOAD** | Chip Select — active-LOW signal that selects one slave device on the bus; pulled LOW before sending, HIGH after |
| **MAX7219** | Serial LED display driver IC — controls up to 64 LEDs (8×8 matrix or 8 seven-segment digits) over SPI at up to 10 MHz |
| **LED matrix** | Grid of LEDs arranged in rows and columns; the MAX7219 drives all 64 with internal multiplexing |
| **Multiplexing** | Technique where the IC activates one row at a time in rapid succession (faster than the eye can detect), creating the illusion that all LEDs are lit simultaneously |
| **Register** | An address inside an IC that stores a setting (e.g. brightness, shutdown mode) or pixel data; written by sending two bytes: address + value |
| **Daisy-chain** | Connecting two or more MAX7219 modules in series (DOUT → DIN) so a single SPI bus drives multiple matrices |
| **Shift register** | Sequential logic circuit that passes data bits from stage to stage on each clock pulse — the basis of SPI communication |
| **ADC** | Analog-to-Digital Converter — converts a voltage (0–3.3 V) into an integer value (0–4095 for 12-bit resolution) |
| **ATTN_11DB** | ADC attenuation setting that extends the measurable range up to ~3.9 V instead of the default ~1.1 V; required to read the full 0–3.3 V output of the KY-018 |
| **LDR** | Light Dependent Resistor (photoresistor) — resistance decreases as light increases; used in the KY-018 module |
| **KY-018** | Photoresistor breakout module — LDR + pull-down resistor forming a voltage divider; brighter light → higher voltage at S pin → higher ADC reading |
| **Voltage divider** | Two resistors in series between power and GND; the output voltage at their junction is proportional to the ratio of their values |
| **VU meter** | Volume Unit meter — a bar whose height represents a signal level; used here as a visual light-level indicator on the LED matrix |
| **I2C** | Inter-Integrated Circuit — a two-wire synchronous serial bus (SDA data + SCL clock) allowing multiple devices on one bus; each device has a unique 7-bit address; open-drain lines require pull-up resistors (typically 4.7 kΩ) |
| **SDA** | I2C serial data line — bidirectional; carries address and data bytes |
| **SCL** | I2C serial clock line — driven by the master (ESP32-S3) to synchronise all transfers |
| **AHT20** | Digital temperature and humidity sensor (Aosong) — I2C address 0x38; triggered by command 0xAC, responds with 6 bytes; temperature −40 to 85 °C ±0.3 °C, humidity 0–100 % ±2 % |
| **BMP280** | Barometric pressure and temperature sensor (Bosch) — I2C address 0x76 (SDO=GND) or 0x77 (SDO=3.3 V); stores 12 factory calibration coefficients that must be read once at startup and applied to every raw ADC reading |
| **hPa** | Hectopascal — SI pressure unit; 1 hPa = 100 Pa; standard sea-level pressure is 1013.25 hPa |
| **Calibration registers** | BMP280 factory values burned into the chip during manufacture; 24 bytes read from address 0x88; the Bosch compensation formulas convert raw ADC counts to accurate temperature and pressure |
| **t_fine** | Intermediate variable in the BMP280 Bosch compensation algorithm: accumulates the compensated temperature and is then reused as an input to the pressure compensation formula |
| **HC-SR04** | Ultrasonic distance sensor — emits a 40 kHz burst and measures the echo travel time to calculate distance (range 2–400 cm) |
| **TRIG** | HC-SR04 trigger pin — a 10 µs HIGH pulse starts one measurement cycle |
| **ECHO** | HC-SR04 echo pin — stays HIGH for a duration proportional to the measured distance; `duration [µs] / 58 ≈ distance [cm]` |
| **time_pulse_us()** | MicroPython function that waits for a pin to reach a given level and measures how long it stays there; built-in timeout prevents infinite loops |
| **TCRT5000** | IR reflective sensor module — an IR LED emits infrared light; a phototransistor detects the reflected light; D0 (digital out) goes LOW when an object is close (active LOW); A0 (analog out) provides a proportional voltage: lower value = more reflection = closer object |
| **IR LED** | Infrared Light-Emitting Diode — emits light at ~950 nm, invisible to the human eye; in the TCRT5000 it illuminates the surface in front of the sensor |
| **Phototransistor** | Light-sensitive transistor — conducts when illuminated by reflected IR light; its conduction level is proportional to the amount of reflected light |
| **RC-522** | RFID reader/writer IC — reads and writes 13.56 MHz MIFARE-compatible cards and tags over SPI; detects cards within ~3 cm |
| **RFID** | Radio Frequency Identification — wireless technology where a reader energises a passive tag via an electromagnetic field; the tag responds with its stored data |
| **MIFARE** | Contactless smartcard standard (13.56 MHz ISO 14443A) from NXP; the most common format for key fobs, transit cards and access control badges |
| **UID** | Unique Identifier — a 4-byte number factory-programmed into each MIFARE card; used to identify individual cards without reading any user data |
| **Anti-collision** | Protocol step where the reader selects one card when multiple are in range simultaneously, preventing data corruption from simultaneous responses |
| **Joystick** | Two-axis potentiometer input device — two variable resistors form voltage dividers for X and Y axes; a push-button switch is built into the shaft |
| **Active LOW** | Digital signal convention where logic 0 (0 V) means the event is active; a button wired active LOW pulls the pin to GND when pressed |
| **Pull-up resistor** | Resistor connected between a signal line and VCC; when the line is floating it reads HIGH, and the signal can pull it LOW without a short circuit |
| **setup()** | Initialisation function called once at program start |
| **loop()** | Main loop function called repeatedly |
| **sleep_ms(n)** | Pauses program execution for *n* milliseconds |
| **REPL** | Read-Eval-Print Loop — the interactive MicroPython console |

## Wiring — MAX7219 8×8 LED matrix (Lesson 13)

```text
ESP32-S3 Pico         MAX7219 module
┌──────────────┐     ┌──────────────────┐
│        GP18  ├─────┤ CLK              │
│        GP11  ├─────┤ DIN              │
│         GP5  ├─────┤ CS               │
│         3V3  ├─────┤ VCC              │
│         GND  ├─────┤ GND              │
│         GP4  │     │ (MISO: not used) │
└──────────────┘     └──────────────────┘
```

> **GP4 / MISO:** The hardware SPI driver on ESP32-S3 requires a MISO pin to be configured even if it is never used. GP4 is assigned as MISO in code but is not connected to the MAX7219 module — leave it floating.
> **VCC voltage:** The MAX7219 module accepts 3.3 V or 5 V. At 3.3 V the maximum LED current is lower, which means slightly less brightness. For most indoor use 3.3 V is sufficient and avoids a level-shifter. If you need maximum brightness, power from the 5 V USB pin and add a 3.3 V-to-5 V level shifter on CLK, DIN and CS.

Practical wiring order:

```text
Step 1: GND   (ESP32-S3)  →  GND   (MAX7219)
Step 2: 3V3   (ESP32-S3)  →  VCC   (MAX7219)
Step 3: GP5   (ESP32-S3)  →  CS    (MAX7219)
Step 4: GP18  (ESP32-S3)  →  CLK   (MAX7219)
Step 5: GP11  (ESP32-S3)  →  DIN   (MAX7219)
        GP4   (ESP32-S3)  →  (nothing)
```

## Wiring — KY-018 photoresistor (Lesson 14)

```text
ESP32-S3 Pico         KY-018 module
┌──────────────┐     ┌──────────────────┐
│         GP7  ├─────┤ S  (signal)      │
│         3V3  ├─────┤ +  (VCC)         │
│         GND  ├─────┤ -  (GND)         │
└──────────────┘     └──────────────────┘
```

> **KY-018 pin order** (looking at the module from the front, pins pointing down): the leftmost pin is **S** (signal/analog out), the middle is **+** (VCC), the rightmost is **-** (GND). Always check the silkscreen on your specific module.
>
> **⚠ Common clone variant — S and GND are swapped:** Many KY-018 modules sold on AliExpress and similar marketplaces have the pin order **− | + | S** (GND on the left, signal on the right) instead of the standard **S | + | −**. If you connect such a module following the diagram above without checking the PCB labels, GND ends up on GP7 and the ADC reads **0** constantly. Always verify the **S**, **+** and **−** markings printed on your specific module's PCB before wiring.

**How it works — voltage divider:**

```text
3.3 V ──── LDR (resistance falls with more light) ──┬──── GP7 (ADC)
                                                     │
                                                 R 10 kΩ (fixed)
                                                     │
                                                    GND
```

More light → LDR resistance decreases → voltage at GP7 rises → higher ADC reading (0–4095).

```text
Ambient light:  │░░░░░░░░░│▒▒▒▒▒▒▒▒▒│█████████│
ADC reading:    0       1000       2500      4095
Category:      DARK     NORMAL      BRIGHT
Bar rows lit:  0–2       3–4        5–8
```

## Wiring — AHT20 + BMP280 (Lesson 15)

Both sensors share the same I2C bus — connect SDA and SCL in parallel.

```text
ESP32-S3 Pico         AHT20 module         BMP280 module
┌──────────────┐     ┌────────────┐        ┌────────────┐
│        GP13  ├─────┤ SDA        ├────────┤ SDA        │
│        GP14  ├─────┤ SCL        ├────────┤ SCL        │
│         3V3  ├─────┤ VCC        │        │ VCC  ──────┤── 3V3
│         GND  ├─────┤ GND        │        │ GND  ──────┤── GND
└──────────────┘     └────────────┘        └────────────┘
```

> **SDO pin on BMP280:** If your module exposes an SDO pin, connect it to GND to fix the I2C address at 0x76 (the default used in the sketch). Connecting SDO to 3.3 V changes the address to 0x77 — update `_BMP280_ADDR` in the code if you do this.
> **Pull-up resistors:** Most breakout modules include 4.7 kΩ pull-ups on SDA and SCL. If you are stacking bare chips on a breadboard without a module, add one 4.7 kΩ resistor from SDA to 3.3 V and another from SCL to 3.3 V.

**Breadboard wiring order:**

```text
Step 1: GND  (ESP32-S3) → GND  (AHT20) and GND  (BMP280)
Step 2: 3V3  (ESP32-S3) → VCC  (AHT20) and VCC  (BMP280)
Step 3: GP13 (ESP32-S3) → SDA  (AHT20) and SDA  (BMP280)
Step 4: GP14 (ESP32-S3) → SCL  (AHT20) and SCL  (BMP280)
```

## Wiring — HC-SR04 ultrasonic distance sensor (Lesson 16)

```text
ESP32-S3 Pico         HC-SR04
┌──────────────┐     ┌─────────┐
│         GP5  ├─────┤ TRIG    │
│         GP4  ├─────┤ ECHO    │
│         3V3  ├─────┤ VCC     │
│         GND  ├─────┤ GND     │
└──────────────┘     └─────────┘
```

> **VCC at 3.3 V:** Powering the HC-SR04 from 3.3 V reduces the ECHO output to ~3.3 V, making it safe to connect directly to GP4. If you power from 5 V, the ECHO output rises to 5 V which exceeds the ESP32-S3's GPIO limit — add a voltage divider (e.g. 1 kΩ + 2 kΩ) or a level shifter on the ECHO line.

**How it works:**

```text
TRIG: ──┐ 10 µs ┌────────────────── LOW
        └───────┘

ECHO: ──────────────┐          ┌───── LOW
                    │← ──────→│
                    │ duration │  = distance × 2 / 343 m/s
                    └──────────┘
      HIGH when echo starts     HIGH ends when echo returns

distance [cm] = duration [µs] / 58
```

## Wiring — TCRT5000 IR reflective sensor (Lesson 17)

```text
ESP32-S3 Pico         TCRT5000 module
┌──────────────┐     ┌─────────┐
│         GP2  ├─────┤ D0      │
│        GP10  ├─────┤ A0      │
│         3.3V ├─────┤ VCC     │
│         GND  ├─────┤ GND     │
└──────────────┘     └─────────┘
```

> **D0 is active LOW:** the digital output goes LOW (0) when an object or reflective surface is detected, and HIGH (1) when the path is clear.
> **A0 analog output:** lower ADC value means more IR reflection (object closer). The ESP32-S3 reads this on GP10 via ADC1 (0–4095, 12-bit).

**Onboard potentiometer:**

```text
TCRT5000 module (top view)
 ┌─────────────────────┐
 │  IR LED   sensor    │
 │                     │
 │  [POT]  [LED]       │   POT = detection threshold
 │                     │       CW → higher threshold (detects from further away)
 │  VCC D0 A0 GND      │       CCW → lower threshold (only very close objects)
 └─────────────────────┘
```

Turn the potentiometer until the onboard indicator LED switches state at the desired detection distance.

## Wiring — PS2 Joystick (Lesson 19)

```text
ESP32-S3 Pico         Joystick module
┌──────────────┐     ┌──────────────┐
│         GP1  ├─────┤ VRx          │
│         GP6  ├─────┤ VRy          │
│         GP8  ├─────┤ SW           │
│         3V3  ├─────┤ VCC          │
│         GND  ├─────┤ GND          │
└──────────────┘     └──────────────┘
```

> **SW pull-up:** The SW button is active LOW — it outputs 0 V when pressed and floats when released. GP8 uses the ESP32-S3 internal pull-up (`Pin.PULL_UP`) so the pin reads 1 at rest and 0 when pressed.
> **Center drift:** The two potentiometers in the joystick rarely settle at exactly 2048. Expect a resting value anywhere between ~1800 and ~2200 — the dead-zone thresholds (< 1000 and > 3000) account for this.

**How the axes work:**

```text
         VRy = 0 (UP)
              │
VRx = 0  ────┼──── VRx = 4095
(LEFT)        │             (RIGHT)
              │
         VRy = 4095 (DOWN)
```

## Communication Protocols

### SPI — Serial Peripheral Interface

SPI is the fastest of the common serial protocols. It is full-duplex (sends and receives simultaneously) and uses **4 wires**. The master (ESP32-S3) controls the clock; each slave device needs its own **CS** (Chip Select) wire.

**Wires:**

| Signal | Alternative name | Description |
| ------ | ---------------- | ----------- |
| MOSI | SDO, DIN, TX | Master Out Slave In — data from master to slave |
| MISO | SDI, DOUT, RX | Master In Slave Out — data from slave to master |
| SCK | CLK | Clock — generated by the master |
| CS | SS, CE, LOAD | Chip Select — one wire per slave, active LOW |

```text
ESP32-S3       MAX7219
  GP18  ──────── CLK
  GP11  ──────── DIN
  GP5   ──────── CS
  GND   ──────── GND
  (MISO not used by MAX7219 — write-only device)
```

**Key rules:**

- No pull-up resistors needed
- Typical speeds: 1–50 MHz (much faster than I2C)
- Each additional slave needs one extra CS pin on the master
- MAX7219 maximum SPI clock: 10 MHz

**How a write transaction works:**

```text
CS:   ─────┐                               ┌────
           └─── LOW (device selected) ─────┘

SCK:  ──┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌──
        └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘

MOSI: ──┬── bit7 ──┬── bit6 ── … ── bit0 ──┬──
        │  (MSB)   │                        │ (LSB)
```

1. CS pulled LOW — MAX7219 starts listening
2. 8 bits of **register address** sent MSB-first
3. 8 bits of **data** sent MSB-first
4. CS pulled HIGH — MAX7219 latches the value

**MicroPython example:**

```python
from machine import SPI, Pin

spi = SPI(1, baudrate=10_000_000, polarity=0, phase=0,
          sck=Pin(18), mosi=Pin(19), miso=Pin(4))
cs = Pin(5, Pin.OUT, value=1)   # CS idle HIGH

cs.value(0)                      # select MAX7219
spi.write(bytes([0x0C, 0x01]))   # write register 0x0C (Shutdown) = 0x01 (normal)
cs.value(1)                      # latch
```

### Protocol comparison

| Feature | UART | I2C | SPI |
| ------- | ---- | --- | --- |
| Wires | 2 (+ GND) | 2 (+ GND) | 4 (+ 1 per slave) |
| Max devices | 2 (point-to-point) | ~127 | unlimited (1 CS per slave) |
| Speed | 115 kbps typical | up to 400 kHz | up to 50 MHz |
| Pull-ups needed | No | Yes (4.7 kΩ) | No |
| Full-duplex | No | No | Yes |
| Complexity | Simplest | Medium | Medium |
| Typical use | Debug, GPS, BT | Sensors, OLED | Displays, SD card |

## MAX7219 Register Reference

The MAX7219 is configured by writing 16-bit frames (address + data). All registers are write-only except No-Op.

| Register | Address | Values | Description |
| -------- | ------- | ------ | ----------- |
| No-Op | `0x00` | any | Does nothing; used for daisy-chain padding |
| Row 1–8 | `0x01–0x08` | `0x00–0xFF` | Raw pixel data — each bit is one LED |
| Decode Mode | `0x09` | `0x00` / `0xFF` | `0x00` = raw pixels; `0xFF` = BCD decode for 7-segment |
| Intensity | `0x0A` | `0x00–0x0F` | Brightness from min (`0x00`) to max (`0x0F`) |
| Scan Limit | `0x0B` | `0x00–0x07` | How many rows to display; `0x07` = all 8 |
| Shutdown | `0x0C` | `0x00` / `0x01` | `0x00` = all LEDs off (low power); `0x01` = normal |
| Display Test | `0x0F` | `0x00` / `0x01` | `0x01` = all LEDs on (ignores other registers) |

**Initialisation sequence:**

```python
_write(0x09, 0x00)   # Decode Mode  : raw pixels
_write(0x0A, 0x04)   # Intensity    : medium (0–15)
_write(0x0B, 0x07)   # Scan Limit   : all 8 rows
_write(0x0C, 0x01)   # Shutdown     : normal operation
_write(0x0F, 0x00)   # Display Test : off
```

### Lesson 13 — MAX7219 8×8 LED matrix

**Goal:** Initialise an SPI peripheral device, write to its internal registers, and build a cyclic pattern animation.

**What happens:**
`setup()` sends five initialisation frames to the MAX7219 over SPI — configuring raw pixel mode, medium brightness, all 8 rows active, normal operation and no test mode. The display is cleared. The `loop()` function cycles through four built-in patterns (Smiley → Heart → Cross → Arrow up → repeat), displays each for 1.5 seconds, and prints the pattern name to the REPL.

**Components used:**

- **MAX7219 8×8 LED matrix module** (common 5-pin breakout: VCC, GND, DIN, CS, CLK)
- 4× jumper wires (male-to-male or male-to-female depending on header type)

**Wiring:**

```text
ESP32-S3 Pico         MAX7219 module
┌──────────────┐     ┌──────────────────┐
│        GP18  ├─────┤ CLK              │
│        GP11  ├─────┤ DIN              │
│         GP5  ├─────┤ CS               │
│         3V3  ├─────┤ VCC              │
│         GND  ├─────┤ GND              │
└──────────────┘     └──────────────────┘
```

**Blockly blocks:**

```text
╔══ ▶ START ════════════════════════════════════════════╗
║  [MAX7219 init  CLK=GP18  DIN=GP11  CS=GP5]           ║
```

**MicroPython code:**

```python
from machine import Pin, I2C
import time

_i2c = I2C(0, sda=Pin(13), scl=Pin(14), freq=400000)
_AHT20_ADDR = 0x38
_BMP280_ADDR = 0x76
_bmp280_cal = None

def _u16(b, i):
    return b[i] | (b[i + 1] << 8)

def _s16(b, i):
    v = _u16(b, i)
    return v - 65536 if v > 32767 else v

def _aht20_init():
    _i2c.writeto(_AHT20_ADDR, bytes([0xBE, 0x08, 0x00]))
    time.sleep_ms(20)

def _aht20_read():
    _i2c.writeto(_AHT20_ADDR, bytes([0xAC, 0x33, 0x00]))
    time.sleep_ms(80)
    d = _i2c.readfrom(_AHT20_ADDR, 6)
    rh = (d[1] << 12) | (d[2] << 4) | (d[3] >> 4)
    rt = ((d[3] & 0x0F) << 16) | (d[4] << 8) | d[5]
    return round(rt / 1048576.0 * 200.0 - 50.0, 1), round(rh / 1048576.0 * 100.0, 1)

def _bmp280_init():
    _i2c.writeto_mem(_BMP280_ADDR, 0xF4, bytes([0xB7]))   # osrs_t=2 osrs_p=5 mode=normal
    time.sleep_ms(10)

def _bmp280_read_cal():
    c = _i2c.readfrom_mem(_BMP280_ADDR, 0x88, 24)
    return (_u16(c,0),_s16(c,2),_s16(c,4),_u16(c,6),_s16(c,8),_s16(c,10),
            _s16(c,12),_s16(c,14),_s16(c,16),_u16(c,18),_s16(c,20),_s16(c,22))

def _bmp280_read(cal):
    r = _i2c.readfrom_mem(_BMP280_ADDR, 0xF7, 6)
    adc_p = (r[0] << 12) | (r[1] << 4) | (r[2] >> 4)
    adc_t = (r[3] << 12) | (r[4] << 4) | (r[5] >> 4)
    v1 = (adc_t / 16384.0 - cal[0] / 1024.0) * cal[1]
    v2 = ((adc_t / 131072.0 - cal[0] / 8192.0) ** 2) * cal[2]
    tf = v1 + v2
    temp = round(tf / 5120.0, 1)
    v1 = tf / 2.0 - 64000.0
    v2 = v1 * v1 * cal[8] / 32768.0 + v1 * cal[7] * 2.0
    v2 = v2 / 4.0 + cal[6] * 65536.0
    v1 = (cal[5] * v1 * v1 / 524288.0 + cal[4] * v1) / 524288.0
    v1 = (1.0 + v1 / 32768.0) * cal[3]
    if v1 == 0:
        return temp, 0.0
    p = 1048576.0 - adc_p
    p = (p - v2 / 4096.0) * 6250.0 / v1
    v1 = cal[11] * p * p / 2147483648.0
    v2 = p * cal[10] / 32768.0
    return temp, round((p + (v1 + v2 + cal[9]) / 16.0) / 100.0, 1)

def setup():
    global _bmp280_cal
    _aht20_init()
    _bmp280_init()
    _bmp280_cal = _bmp280_read_cal()
    print('AHT20 + BMP280 ready   SDA=GP13  SCL=GP14')

def loop():
    temp_a, hum = _aht20_read()
    temp_b, pres = _bmp280_read(_bmp280_cal)
    print('AHT20:  Temp=' + str(temp_a) + ' C   Hum=' + str(hum) + ' %')
    print('BMP280: Temp=' + str(temp_b) + ' C   Pres=' + str(pres) + ' hPa')
    time.sleep_ms(2000)
```

**Example REPL output:**

```text
AHT20 + BMP280 ready   SDA=GP13  SCL=GP14
AHT20:  Temp=22.4 C   Hum=51.3 %
BMP280: Temp=22.7 C   Pres=1013.5 hPa
AHT20:  Temp=22.4 C   Hum=51.2 %
BMP280: Temp=22.7 C   Pres=1013.4 hPa
```

> **Temperature difference:** AHT20 and BMP280 measure temperature independently. A small difference of 0.5–2 °C between them is normal — they use different sensing elements and have different self-heating characteristics.

**What you learn:**

- Initialising `machine.I2C` and communicating with two devices on the same bus
- Sending write commands to an I2C sensor and reading back a response
- Reading and parsing BMP280 factory calibration data (24 bytes, mix of signed and unsigned 16-bit words)

```python
_echo = Pin(4, Pin.IN)
```

```python
import time

_spi_r = SPI(1, baudrate=1_000_000, polarity=0, phase=0, sck=Pin(18), mosi=Pin(11), miso=Pin(16))
_cs_r  = Pin(17, Pin.OUT, value=1)
_rst_r = Pin(15, Pin.OUT, value=1)

def _rrd(a):
    _cs_r.value(0); _spi_r.write(bytes([((a<<1)&0x7E)|0x80])); v=_spi_r.read(1)[0]; _cs_r.value(1); return v
def _rwr(a,v):
    _cs_r.value(0); _spi_r.write(bytes([(a<<1)&0x7E,v])); _cs_r.value(1)
def _rset(a,m): _rwr(a,_rrd(a)|m)
def _rclr(a,m): _rwr(a,_rrd(a)&~m)

def _transceive(data):
    _rwr(0x02,0x77); _rclr(0x04,0x80); _rset(0x0A,0x80); _rwr(0x01,0x00)
    for b in data: _rwr(0x09,b)
    _rwr(0x01,0x0C); _rset(0x0D,0x80)
    for _ in range(2000):
        if _rrd(0x04)&0x31: break
    _rclr(0x0D,0x80)
    if _rrd(0x06)&0x1B: return None
    n=_rrd(0x0A)&0x7F; return [_rrd(0x09) for _ in range(min(n,16))]

def read_uid():
    _rwr(0x0D,0x07)
    r=_transceive([0x26])
    if not r or len(r)<2: return None
    _rwr(0x0D,0x00)
    r=_transceive([0x93,0x20])
    if r and len(r)==5 and (r[0]^r[1]^r[2]^r[3])==r[4]:
        return ':'.join('{:02X}'.format(b) for b in r[:4])
    return None

def setup():
    _rst_r.value(0); time.sleep_ms(1); _rst_r.value(1); time.sleep_ms(50)
    _rwr(0x01,0x0F); time.sleep_ms(50)
    _rwr(0x2A,0x8D); _rwr(0x2B,0x3E); _rwr(0x2C,30); _rwr(0x2D,0)
    _rwr(0x15,0x40); _rwr(0x11,0x3D); _rset(0x14,0x03)
    print('RC-522 ready   SCK=GP18 MOSI=GP11 MISO=GP16 SDA=GP17 RST=GP15')
    print('Hold a card or tag near the reader...')

def loop():
    uid = read_uid()
    if uid:
        print('Card UID: ' + uid)
        time.sleep_ms(1000)
    else:
        time.sleep_ms(100)
```

```text
Card UID: A3:1F:C7:02
Card UID: 7B:44:E9:1A
```


**MicroPython code:**

```python
from machine import Pin, ADC
import time

_adc_x = ADC(Pin(1), atten=ADC.ATTN_11DB)
_adc_y = ADC(Pin(6), atten=ADC.ATTN_11DB)
_sw    = Pin(8, Pin.IN, Pin.PULL_UP)

def read_joystick():
    return _adc_x.read(), _adc_y.read(), not _sw.value()

def joy_direction(x, y):
    h = 'LEFT'  if x < 1000 else ('RIGHT' if x > 3000 else '')
    v = 'UP'    if y < 1000 else ('DOWN'  if y > 3000 else '')
    if h and v: return h + '-' + v
    return h or v or 'CENTER'

def setup():
    print('Joystick ready   VRx=GP1  VRy=GP6  SW=GP8')

def loop():
    x, y, btn = read_joystick()
    direction  = joy_direction(x, y)
    btn_str    = ' [BTN]' if btn else ''
    print('X:' + str(x) + '  Y:' + str(y) + '  ' + direction + btn_str)
    time.sleep_ms(200)
```

**Example REPL output** (move the joystick in different directions, then press the button):

```text

X:2048  Y:2051  CENTER
X:412   Y:2051  LEFT
X:412   Y:389   UP-LEFT
X:2048  Y:3891  DOWN
X:2048  Y:2048  CENTER [BTN]
```

**What you learn:**

- Reading two analog channels simultaneously with `machine.ADC`
- Mapping analog values to discrete zones with threshold comparisons
- Reading an active-LOW digital button with `Pin.PULL_UP`
- Combining multiple sensor values into a compact human-readable print line
- Using Python's `or` chaining (`h or v or 'CENTER'`) as a concise conditional expression

## Program structure

Every sketch in this course follows the same pattern:

```python
from machine import Pin   # hardware module imports
import time               # time module

# --- Hardware configuration (global) ---
_pin_XX = Pin(XX, mode=Pin.OUT)

# --- One-time initialisation ---
def setup():
    pass  # runs once at startup

# --- Main loop ---
def loop():
    pass  # runs repeatedly

# --- Entry point ---
if __name__ == '__main__':
    try:
        setup()
        while True:
            loop()
    except (Exception, KeyboardInterrupt) as e:
        print(e)
```

> The `try/except` block at the end ensures that pressing Ctrl+C in the REPL console safely stops the program instead of raising an unhandled exception.

### Interactive REPL — testing code line by line

After the sketch finishes (or after pressing `Ctrl + C`), the `>>>` prompt appears. You can test individual commands:

```text
>>> from machine import SPI, Pin
>>> spi = SPI(1, baudrate=10_000_000, polarity=0, phase=0, sck=Pin(18), mosi=Pin(19), miso=Pin(4))
>>> cs = Pin(5, Pin.OUT, value=1)
>>> cs.value(0)
>>> spi.write(bytes([0x0F, 0x01]))   # Display Test: all LEDs on

>>> cs.value(0)
>>> spi.write(bytes([0x0F, 0x00]))   # Display Test: off
```

This is useful for:

- **Verifying wiring** — display test lights all LEDs instantly
- **Testing a single register write** before putting it in a sketch
- **Debugging** — check what value a register produces

### Troubleshooting

| Symptom | Likely cause | Fix |
| ------- | ------------ | --- |
| No LEDs light up | MAX7219 in shutdown mode or wrong CS polarity | Check that init_max7219() runs; verify GP5 → CS wiring |
| All LEDs permanently on | Display Test register left at 0x01 | Send `_write(0x0F, 0x00)` to exit test mode |
| Random garbage on display | Wrong SPI polarity/phase | MAX7219 uses `polarity=0, phase=0` (SPI mode 0) |
| KY-018 ADC always reads 0 | S and GND pins swapped on clone module (pin order − \| + \| S instead of S \| + \| −) | Check the silkscreen on the module PCB; rewire so that the pin labelled **S** goes to GP7 |
| Port not found | USB cable not connected or charge-only cable | Use a data USB cable |
| `OSError: [Errno 16]` | Port busy | Close Thonny, mpremote, or other serial monitors |
| Upload hangs | Board stuck in running program | Press `Ctrl + C` in terminal first, then upload |
