# MicroPython Course — ESP32-S3 Pico · Part 3

The same seven sensors from Part 2 — but all output now goes to an **LCD1602 I2C display** (SDA=GP21, SCL=GP22) instead of the serial terminal. Each lesson is self-contained: wire the LCD plus the lesson's sensor, open the sketch, run it.

## Glossary

| Term | Description |
| ---- | ----------- |
| **GPIO** | General Purpose Input/Output — a microcontroller pin that can act as a digital input or output |
| **I2C** | Inter-Integrated Circuit — two-wire serial bus (SDA = data, SCL = clock) for short-range communication between ICs on the same board |
| **SDA** | I2C data line — carries bits bidirectionally between master and slave |
| **SCL** | I2C clock line — generated by the master to synchronise transfers |
| **PCF8574** | I2C GPIO expander IC soldered to the back of most LCD1602 modules; converts 8 parallel LCD control signals to 2-wire I2C |
| **LCD1602** | 16-column × 2-row character LCD; the PCF8574 backpack lets it be driven over I2C with only SDA + SCL |
| **write_line()** | Custom driver method that positions the cursor on a row and writes a string padded/truncated to 16 characters |
| **SPI** | Serial Peripheral Interface — fast 4-wire bus (SCK, MOSI, MISO, CS) used by the MAX7219 and RC-522 |
| **ADC** | Analog-to-Digital Converter — converts a voltage (0–3.3 V) into an integer 0–4095 |
| **ATTN_11DB** | ADC attenuation that extends the readable range to ~3.9 V; required to cover the full 0–3.3 V output of resistive sensors |
| **MAX7219** | SPI LED-matrix driver controlling up to 64 LEDs over 3 wires |
| **KY-018** | Photoresistor breakout; brighter light → higher ADC reading |
| **DHT11** | Digital temperature & humidity sensor using a single-wire protocol |
| **HC-SR04** | Ultrasonic distance sensor (2–400 cm) using TRIG + ECHO pins |
| **HC-SR501** | PIR passive-infrared motion sensor; OUT goes HIGH on detected motion |
| **RC-522** | 13.56 MHz MIFARE RFID reader/writer over SPI |
| **UID** | Unique Identifier — 4-byte number factory-programmed into each MIFARE card |
| **Joystick** | Two ADC axes (X/Y) + active-LOW push button for 8-directional input |
| **Active LOW** | A signal that is "on" when the voltage is 0 V; button wired with pull-up reads HIGH at rest, LOW when pressed |
| **setup()** | Initialisation function called once at program start |
| **loop()** | Main loop function called repeatedly |
| **sleep_ms(n)** | Pauses execution for *n* milliseconds |

## Wiring — LCD1602 I2C (all lessons)

Connect the LCD1602 before adding any sensor — it is required by all lessons.

```text
ESP32-S3 Pico         LCD1602 I2C backpack
┌──────────────┐     ┌──────────────────────┐
│        GP21  ├─────┤ SDA                  │
│        GP22  ├─────┤ SCL                  │
│         3V3  ├─────┤ VCC                  │
│         GND  ├─────┤ GND                  │
└──────────────┘     └──────────────────────┘
```

> **I2C address:** Most PCF8574 backpacks ship with address `0x27`. If the display stays blank, try `0x3F`. You can scan with `I2C(0, sda=Pin(21), scl=Pin(22)).scan()`.

Practical wiring order:

```text
Step 1: GND  (ESP32-S3)  →  GND  (LCD1602)
Step 2: 3V3  (ESP32-S3)  →  VCC  (LCD1602)
Step 3: GP21 (ESP32-S3)  →  SDA  (LCD1602)
Step 4: GP22 (ESP32-S3)  →  SCL  (LCD1602)
```

## Wiring — HC-SR501 PIR motion sensor (Lesson 17)

```text
ESP32-S3 Pico         HC-SR501 module
┌──────────────┐     ┌──────────────────┐
│         GP2  ├─────┤ OUT              │
│        VBUS  ├─────┤ VCC  (5 V only!) │
│         GND  ├─────┤ GND              │
└──────────────┘     └──────────────────┘
```

> **IMPORTANT: HC-SR501 requires 5 V on VCC.** Use the VBUS pin on the ESP32-S3 Pico (which exposes the USB 5 V rail). Powering from 3.3 V will cause unstable or no output.

Practical wiring order:

```text
Step 1: GND   (ESP32-S3)  →  GND  (HC-SR501)
Step 2: VBUS  (ESP32-S3)  →  VCC  (HC-SR501)
Step 3: GP2   (ESP32-S3)  →  OUT  (HC-SR501)
```

Allow ~30 seconds after power-on for the sensor to stabilise before expecting reliable motion detection.

## Lesson 13 — MAX7219 8×8 LED matrix with LCD

**Goal:** cycle through four pixel patterns on the LED matrix and show the pattern name on the LCD1602.

**What happens:**

- Setup: initialises MAX7219, clears the matrix, shows `MAX7219 + LCD` / `Ready` on the LCD.
- Loop: shows SMILEY → HEART → CROSS → ARROW UP on the matrix; writes `Pattern:` to LCD line 1 and the name to LCD line 2, 1.5 s per step.

**Components:** ESP32-S3 Pico · MAX7219 module · LCD1602 I2C

**Wiring reference:** MAX7219 + LCD1602 sections above.

**Blockly overview:**

```
[ Setup ]
  max7219_init → max7219_clear → lcd_init → lcd_write(0,"MAX7219 + LCD") → lcd_write(1,"Ready")

[ Loop forever ]
  max7219_show(SMILEY)   → lcd_write(0,"Pattern:") → lcd_write(1,"Smiley")   → sleep 1500
  max7219_show(HEART)    → lcd_write(0,"Pattern:") → lcd_write(1,"Heart")    → sleep 1500
  max7219_show(CROSS)    → lcd_write(0,"Pattern:") → lcd_write(1,"Cross")    → sleep 1500
  max7219_show(ARROW_UP) → lcd_write(0,"Pattern:") → lcd_write(1,"Arrow up") → sleep 1500
```

**MicroPython code:**

```python
from machine import Pin, SPI, I2C
from lcd1602 import LCD1602
import time

_spi = SPI(1, baudrate=10_000_000, polarity=0, phase=0,
           sck=Pin(18), mosi=Pin(19), miso=Pin(4))
_cs  = Pin(5, Pin.OUT, value=1)
_i2c = I2C(0, sda=Pin(21), scl=Pin(22), freq=400000)
_lcd = LCD1602(_i2c, 0x27)

# ... init_max7219(), show_pattern(), patterns ...

def setup():
    init_max7219(); clear_display()
    _lcd.clear()
    _lcd.write_line(0, 'MAX7219 + LCD')
    _lcd.write_line(1, 'Ready')

def loop():
    global _step
    show_pattern(_patterns[_step])
    _lcd.write_line(0, 'Pattern:')
    _lcd.write_line(1, _names[_step])
    _step = (_step + 1) % len(_patterns)
    time.sleep_ms(1500)
```

**What you learn:** I2C LCD initialisation, writing to specific LCD rows, combining SPI and I2C peripherals in one sketch.

## Lesson 14 — KY-018 light sensor with LCD

**Goal:** read ambient light level and display the raw ADC value and category on the LCD1602.

**What happens:**

- Setup: initialises LCD, shows `KY-018 ready` / `ADC=GP7`.
- Loop: reads raw ADC (0–4095), writes `Light: <value>` on line 1, `DARK` / `NORMAL` / `BRIGHT` on line 2 every 500 ms.

**Components:** ESP32-S3 Pico · KY-018 module · LCD1602 I2C

**Wiring reference:** KY-018 + LCD1602 sections above.

**Blockly overview:**

```
[ Setup ]
  lcd_init → lcd_write(0,"KY-018 ready") → lcd_write(1,"ADC=GP7")

[ Loop forever ]
  raw = ky018_read
  lcd_write(0, "Light: " + raw)
  if raw < 1000  → lcd_write(1,"DARK")
  elif raw < 2500 → lcd_write(1,"NORMAL")
  else            → lcd_write(1,"BRIGHT")
  sleep 500
```

**MicroPython code:**

```python
from machine import Pin, ADC, I2C
from lcd1602 import LCD1602
import time

_adc = ADC(Pin(7), atten=ADC.ATTN_11DB)
_i2c = I2C(0, sda=Pin(21), scl=Pin(22), freq=400000)
_lcd = LCD1602(_i2c, 0x27)

def setup():
    _lcd.clear()
    _lcd.write_line(0, 'KY-018 ready')
    _lcd.write_line(1, 'ADC=GP7')

def loop():
    raw = _adc.read()
    _lcd.write_line(0, 'Light: ' + str(raw))
    if   raw < 1000:  _lcd.write_line(1, 'DARK')
    elif raw < 2500:  _lcd.write_line(1, 'NORMAL')
    else:             _lcd.write_line(1, 'BRIGHT')
    time.sleep_ms(500)
```

**LCD output examples:**

```

NORMAL              DARK                BRIGHT
```

**What you learn:** reading ADC on ESP32-S3, category logic, updating specific LCD rows independently.

## Lesson 16 — HC-SR04 ultrasonic distance with LCD

**Goal:** measure distance with ultrasound and display the result on the LCD1602.

**What happens:**

- Setup: initialises LCD, shows `HC-SR04 ready` / `TRIG=GP5 ECHO=GP4`.
- Loop: triggers measurement, writes `Distance:` on line 1 and the distance in cm (or `Out of range`) on line 2 every 500 ms.

**Components:** ESP32-S3 Pico · HC-SR04 sensor · LCD1602 I2C

**Wiring reference:** HC-SR04 + LCD1602 sections above.

**Blockly overview:**

```
[ Setup ]
  lcd_init → lcd_write(0,"HC-SR04 ready") → lcd_write(1,"TRIG=GP5 ECHO=GP4")

[ Loop forever ]
  dist = hcsr04_measure
  lcd_write(0, "Distance:")
  if dist < 2 or dist > 400  → lcd_write(1,"Out of range")
  else                        → lcd_write(1, dist + " cm")
  sleep 500
```

**MicroPython code:**

```python
from machine import Pin, time_pulse_us, I2C
from lcd1602 import LCD1602
import time

_trig = Pin(5, Pin.OUT); _echo = Pin(4, Pin.IN)
_i2c  = I2C(0, sda=Pin(21), scl=Pin(22), freq=400000)
_lcd  = LCD1602(_i2c, 0x27)

def measure_distance():
    _trig.value(0); time.sleep_us(2)
    _trig.value(1); time.sleep_us(10); _trig.value(0)
    dur = time_pulse_us(_echo, 1, 30_000)
    return -1 if dur < 0 else dur / 58

def loop():
    dist = measure_distance()
    _lcd.write_line(0, 'Distance:')
    if dist < 0 or dist < 2 or dist > 400:
        _lcd.write_line(1, 'Out of range')
    else:
        _lcd.write_line(1, str(round(dist, 1)) + ' cm')
    time.sleep_ms(500)
```

**LCD output examples:**

```
Distance:           Distance:
14.2 cm             Out of range
```

**What you learn:** echo-pulse timing, range validation, updating line 2 conditionally.

## Lesson 18 — RC-522 RFID reader with LCD

**Goal:** read RFID card UIDs and display them on the LCD1602.

**What happens:**

- Setup: initialises RC-522 and LCD, shows `RFID reader` / `Hold card...`.
- Loop: polls for a card. When detected: writes `UID:` on line 1 and the hex UID on line 2, pauses 1 s. Otherwise shows `Hold card...` on line 2 and polls every 100 ms.

**Components:** ESP32-S3 Pico · RC-522 module · LCD1602 I2C · RFID card or fob

**Wiring reference:** RC-522 + LCD1602 sections above.

**Blockly overview:**

```
[ Setup ]
  rc522_init → lcd_init → lcd_write(0,"RFID reader") → lcd_write(1,"Hold card...")

[ Loop forever ]
  uid = rc522_read_uid
  if uid != ""
    lcd_write(0,"UID:") → lcd_write(1, uid) → sleep 1000
  else
    lcd_write(1,"Hold card...") → sleep 100
```

**MicroPython code:**

```python
from machine import Pin, I2C
from lcd1602 import LCD1602
import time, mfrc522

_reader = mfrc522.MFRC522(sck=18, mosi=19, miso=16, rst=15, cs=17)
_i2c    = I2C(0, sda=Pin(21), scl=Pin(22), freq=400000)
_lcd    = LCD1602(_i2c, 0x27)

def read_uid():
    stat, _ = _reader.request(_reader.REQIDL)
    if stat != _reader.OK: return None
    stat, uid = _reader.SelectTagSN()
    if stat != _reader.OK: return None
    return ':'.join('{:02X}'.format(b) for b in uid)

def loop():
    uid = read_uid()
    if uid:
        _lcd.write_line(0, 'UID:')
        _lcd.write_line(1, uid)
        time.sleep_ms(1000)
    else:
        _lcd.write_line(1, 'Hold card...')
        time.sleep_ms(100)
```

**LCD output examples:**

```
RFID reader         UID:
Hold card...        A3:2F:B7:91
```

**What you learn:** two-step RFID detection (REQIDL → SelectTagSN), hex UID formatting, conditional LCD line 1 update.

```python
from lcd1602 import LCD1602
import time

_adc_x = ADC(Pin(1), atten=ADC.ATTN_11DB)
_adc_y = ADC(Pin(6), atten=ADC.ATTN_11DB)
_sw    = Pin(8, Pin.IN, Pin.PULL_UP)
_i2c   = I2C(0, sda=Pin(21), scl=Pin(22), freq=400000)
_lcd   = LCD1602(_i2c, 0x27)

def joy_direction(x, y):
    h = 'LEFT' if x < 1000 else ('RIGHT' if x > 3000 else '')
    v = 'UP'   if y < 1000 else ('DOWN'  if y > 3000 else '')
    if h and v: return h + '-' + v
    return h or v or 'CENTER'

def loop():
    x, y, btn = _adc_x.read(), _adc_y.read(), not _sw.value()
    _lcd.write_line(0, 'X:' + str(x) + ' Y:' + str(y))
    _lcd.write_line(1, joy_direction(x, y) + (' [BTN]' if btn else ''))
    time.sleep_ms(200)
```

**LCD output examples:**

```
X:2048 Y:2048       X:234  Y:2049       X:2048 Y:2048
CENTER              LEFT                CENTER [BTN]
```

**What you learn:** dual ADC axes, active-LOW button with pull-up, fitting position data into 16 LCD characters.
